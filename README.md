# dict

A local Dutch -> English dictionary reader built from `van-dale.mobi`.

## Architecture

```
van-dale.mobi
     |
     v
[extractor] (scripts/extract.py)
     |
     v
data/van-dale.sqlite  (SQLite + FTS5)
     |
     v
[server] (server/)  -- Hono, reads SQLite
     |
     v
[frontend] (src/)  -- Svelte SPA
```

### Why SQLite instead of JSON chunks?

The dictionary is ~23 MB. Splitting into JSON files by letter is awkward
to maintain and ships everything to the client. SQLite with FTS5 gives us:

- Fast prefix and full-text search with a single query
- A single file artifact instead of 26+ JSON files
- No chunking strategy to maintain
- Easy to regenerate from the MOBI at any time

The backend is a thin Hono server. It exists only to query SQLite - there is
no business logic.

## Database schema

Two tables, generated by `scripts/extract.py`:

**`entries`** - one row per `<idx:entry>` block in the MOBI (113,194 total)

| Column | Type | Notes |
|--------|------|-------|
| `id` | integer | primary key |
| `headword` | text | Dutch word, e.g. `fiets` |
| `homonym_num` | integer | 1/2/3 when the same headword has multiple entries (e.g. `lopen¹`, `lopen²`) |
| `pos` | text | part of speech abbreviation, e.g. `zelfst nw`, `ov ww` |
| `article` | text | `de` or `het` (nouns only) |
| `gender` | text | `m` or `v` (nouns only, where marked) |
| `inflections` | text | JSON array of inflected forms - plurals, conjugations |
| `body_html` | text | full definition HTML, rendered as-is in the frontend |

**`phrases`** - one row per `◦` fixed expression (5,746 total), linked to parent entry

| Column | Type | Notes |
|--------|------|-------|
| `id` | integer | primary key |
| `entry_id` | integer | FK to `entries` |
| `dutch` | text | Dutch phrase text, e.g. `dat komt duur te staan` |
| `translation` | text | English translation |
| `body_html` | text | full phrase HTML |

FTS5 virtual tables `entries_fts` (headword + inflections) and `phrases_fts`
(dutch) provide fast prefix and token search across both tables.

### TypeScript types

Generated from the live database by `kysely-codegen`. Run `npm run types:gen`
to regenerate after a schema change. Config and notes on FTS5 shadow tables
are in `kysely-codegen.config.ts`.

POS abbreviations are mapped to unabbreviated Dutch in `src/lib/pos.ts`.

## Deployment

A `Dockerfile` builds a single production image:

1. Builds the Svelte frontend (`vite build`)
2. Runs the extractor to generate `data/van-dale.sqlite`
3. Starts the Hono server, which serves the built frontend as static files
   in addition to the API endpoints

This means one container, one port - no separate static hosting needed.

## Dev setup

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

npm install
npm run data:make   # extract MOBI -> data/van-dale.sqlite (run once)
npm run dev:server  # start Hono server
npm run dev         # start Vite dev server
```

## Scripts

| Command | Description |
|---------|-------------|
| `npm run data:make` | Run Python extractor: MOBI -> `data/van-dale.sqlite` |
| `npm run test:extract` | Run extractor unit tests |
| `npm run types:gen` | Regenerate `src/lib/db.d.ts` from the SQLite schema |
| `npm run dev` | Start Vite dev server (frontend) |
| `npm run dev:server` | Start Hono server (backend) |
| `npm run build` | Build frontend for production |
| `npm run check` | TypeScript + Svelte type check |

## Dependencies

**Python (extractor, venv at `.venv/`)**

| Package | Purpose |
|---------|---------|
| `mobi` | Extract HTML from MOBI file |
| `beautifulsoup4` + `lxml` | Parse entry HTML |
| `sqlite3` | Write database (stdlib) |
| `pytest` | Extractor tests |

**Node (server + frontend)**

| Package | Purpose |
|---------|---------|
| `hono` | Backend server |
| `better-sqlite3` | SQLite driver (sync, fast) |
| `kysely-codegen` | Generate TypeScript types from SQLite schema |

## File layout

```
dict/
  scripts/
    extract.py           # MOBI -> SQLite extractor (Python)
    test_extract.py      # extractor unit tests
  server/
    index.ts             # Hono server
  src/
    lib/
      db.d.ts            # generated TypeScript types (kysely-codegen)
      pos.ts             # POS abbreviation -> Dutch label mapping
  data/
    van-dale.sqlite      # generated, not committed
  requirements.txt       # Python dependencies
  kysely-codegen.config.ts
  sources/
    van-dale.mobi        # source dictionary, not committed
```
